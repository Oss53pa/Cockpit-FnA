# Kit beta privée Cockpit FnA

Ce dossier contient tout ce qu'il faut pour orchestrer une beta privée de 4 semaines avec 3-5 clients pilotes.

## Ordre de lecture

1. **`PLAN_BETA_4_SEMAINES.md`** — vue d'ensemble et timeline jour par jour
2. **`../BETA_LAUNCH_CHECKLIST.md`** — checklist technique à valider avant J0
3. **`EMAIL_INVITATION_PILOTE.md`** — template à personnaliser et envoyer aux prospects
4. **`NDA_PILOTE_TEMPLATE.md`** — accord léger à faire signer avant les accès
5. **`EMAIL_KICKOFF_PILOTE.md`** — template envoyé après acceptation, avec les accès

## Action immédiate

1. Lire `PLAN_BETA_4_SEMAINES.md` en entier (15 min)
2. Compléter `../BETA_LAUNCH_CHECKLIST.md` section §0 (pré-requis techniques)
3. Identifier 8-10 prospects (l'objectif est 5 acceptations)
4. Personnaliser et envoyer `EMAIL_INVITATION_PILOTE.md`

## Critères Go/No-Go pour GA (résumé)

🟢 **GO** si :
- NPS moyen ≥ 7
- ≥ 60% des pilotes prêts à payer
- 0 bug critique non résolu
- Docs utilisateur publiables

🔴 **NO-GO** si :
- NPS < 5
- < 3 pilotes ont vraiment utilisé l'outil
- Bug critique persistant

Cf. `PLAN_BETA_4_SEMAINES.md` pour les critères détaillés.

## Ce que je ne peux PAS faire à votre place (Claude)

- ❌ Recruter les pilotes (c'est votre réseau)
- ❌ Conduire les onboardings visio
- ❌ Répondre aux pilotes dans le canal pilote
- ❌ Itérer pendant 4 semaines

## Ce que je PEUX faire pendant la beta

- ✅ Implémenter les fixes prioritaires détectés par les pilotes
- ✅ Analyser les feedbacks compilés et suggérer des décisions roadmap
- ✅ Préparer les annonces produit hebdo aux pilotes
- ✅ Préparer le pricing GA basé sur les retours pilotes
- ✅ Préparer la page de lancement GA + funnel d'acquisition

Lancez la beta, partagez-moi les feedbacks bruts chaque semaine, je vous aide à itérer.
